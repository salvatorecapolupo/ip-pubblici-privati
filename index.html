Differenza tra Indirizzi IP Pubblici e Privati

Guida tecnica basata sugli standard IETF e architetture di rete Cisco

1. IP Pubblico e Privato: Definizioni

In una rete informatica, l'indirizzo IP funge da identificativo univoco. Tuttavia, per gestire la scarsità di indirizzi IPv4, è stata introdotta la distinzione tra ambiti pubblici e privati.

  • IP Pubblico: Raggiungibile globalmente su Internet. È assegnato dal tuo ISP e identifica il tuo router all'esterno.
  • IP Privato: Utilizzato solo all'interno della LAN (casa o ufficio). Non è visibile dall'esterno grazie al protocollo NAT (Network Address Translation).

2. Gli standard RFC 1918

La documentazione ufficiale definisce tre intervalli specifici per le reti private che non devono mai essere instradati sui nodi di backbone di Internet:

Classe Intervallo IP Prefisso CIDR
Classe A 10.0.0.0 - 10.255.255.255 10.0.0.0/8
Classe B 172.16.0.0 - 172.31.255.255 172.16.0.0/12
Classe C 192.168.0.0 - 192.168.255.255 192.168.0.0/16

3. Architettura DMZ (Demilitarized Zone)

La DMZ è strutturalmente una sottorete isolata. Ecco come gestisce gli IP:

  • Server in DMZ: Possiedono un IP privato (es. 172.16.1.10), ma il Firewall esegue un Static NAT associandolo a un IP pubblico univoco per permettere agli utenti esterni di raggiungere il sito web.
  • Sicurezza: Se un server in DMZ viene compromesso, il firewall impedisce all'attaccante di saltare (pivot) verso la rete interna (LAN), dove risiedono i dati sensibili.

In uno scenario Cisco Packet Tracer, vedresti un router con tre interfacce attive, ognuna con un gateway appartenente a subnet diverse.
